Ce document définit les fonctionnalités attendues pour l'application web de gestion de projet agile
Sprint 1 : Développement du cœur de l'application
User Story 1 : En tant qu' utilisateur, je veux voir un tableau avec trois colonnes
(To do, In progress, Done) afin d' organiser visuellement mon travail.
Critères d'acceptation : Les trois colonnes sont présentes et clairement nommées.
User Story 2: En tant que Product Owner, je veux ajouter une User Story avec un titre et une description afin de remplir mon backlog
Critères d'acceptation : Un formulaire permet la saisie et le ticket s'affiche dans la colonne "To do".
User Story 3 : En tant qu' utilisateur, je veux modifier ou supprimer une User Story afin de mettre à jour les besoins du projet.
Critères d'acceptation : Les options de modification et de suppression sont fonctionnelles pour chaque ticket.
User Story 4 :En tant qu' utilisateur, je veux déplacer une User Story entre les colonnes afin de suivre l'avancement des tâches.
Critères d'acceptation : Le statut du ticket change lorsqu'il est déplacé d'une colonne à une autre.
Sprint 2 : Intégration de l'IA
User Story 5 : En tant que Product Owner, je veux que l'IA génère des critères d'acceptation pour une User Story donnée
User Story 6 : En tant qu' utilisateur, je veux que l'IA propose une estimation chiffrée (1, 3, 5, 8, 13) pour chaque ticket
User Story 7 : En tant qu' utilisateur, je veux que l'IA suggère une priorité (Haute, Moyenne, Faible) pour les tickets.
